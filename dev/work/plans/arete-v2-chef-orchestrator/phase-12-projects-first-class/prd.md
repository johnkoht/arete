# PRD — Phase 12: Projects as first-class citizens (gated slices A+B+C)

> Generated by ship Phase 2.2 from `plan.md` (incl. **Amendment — 2026-06-10**, which governs) + `pre-mortem.md` + `review.md` (approve-with-suggestions; all 5 suggestions folded in below).
> **Scope**: AC1, AC2, AC3, AC4, AC6, AC10, AC11, AC12 ONLY. AC5/AC7/AC8/AC9 are deferred — do not build, do not scaffold.

## Goal

Make a project's `area` a reliably-present, system-derived field so the already-shipped project brief lights up (AC1), give honest visibility when it can't be derived (AC6), provide an approval-gated fleet backfill (AC2), and ship the read-only `/project` open experience with a topic-aware, project-grained brief (AC3+AC4). Hard gate: `arete brief --project glance-2-mvp` section count 2 → 4+ **and** `metadata.area === "glance-2-mvp"` after Slice A.

## Memory synthesis (Phase 2.1 — binding on all tasks)

1. Dark-code audit is a distinct gate: every new export needs a production call-site (recurring failure mode — topic-wiki 2026-04-23).
2. Run the FULL test suite before wrap; per-file `tsx --test` is blind to cross-file conflicts (2026-06-08).
3. No direct `fs` in services — `StorageAdapter` only (`getModified()` covers mtime). Tests for memory/storage ops use real fs + `FileStorageAdapter` on unique temp dirs (`pid+Date.now()`), no mocks.
4. CLI conventions: `findRoot()` guard, `--json` complete in ALL exit paths, `formatters.ts` helpers, register in `index.ts`, routability + `.agents/sources` docs.
5. Per-task commits; sequential execution; pre-mortem mitigations embedded per task.

## Pre-mortem mitigations in force (amended plan)

- **R1/R2**: eliminated by scope cut — nothing in this build writes project READMEs except approval-gated `backfill-area --apply` (fixture-tested only; live apply is post-merge, John-operated).
- **R3**: confidence floor 0.7 non-negotiable; preview default; `area_set_by: backfill` provenance; below floor → propose nothing.
- **R4**: parser tolerates a future `areas:` list (first entry) without building plural support.
- **R5**: never auto-load a tie — top-N disambiguation.
- **R6**: creation-time area proposal is optional with default + skip; never blocks creation.
- **R9**: frontmatter wins on fm/prose disagreement; divergence surfaced as a one-line warning.
