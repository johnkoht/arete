## Plan: Split Sync Skill into Focused Integration Skills

**Status**: draft (reviewed)  
**Size**: Medium-Large (6 steps)  
**Work Type**: Refactor + New Feature

### Problem Statement

The `sync` skill is a 400+ line monolith trying to document 5 different integrations (Fathom, Krisp, Notion, Calendar, Slack). This creates:
- **Routing confusion** - triggers overlap, agent loads irrelevant guidance
- **Maintenance burden** - changing one integration risks breaking others
- **Context bloat** - user asking about Fathom loads Slack/Notion/Calendar docs they don't need

### Success Criteria

1. Each integration has its own focused skill (~60-100 lines)
2. Meeting integrations (Fathom, Krisp) have their own templates
3. Templates produce output compatible with `process-meetings` skill
4. Name enrichment from calendar is documented as a pattern
5. `sync` skill is deleted
6. CLI (`arete pull [integration]`) unchanged

### Architecture Decision: Two-Stage Flow

**Clarification**: This plan implements a two-stage architecture:

```
Stage 1: arete pull fathom    → Core adapter writes meeting file (hardcoded template)
Stage 2: process-meetings     → Transforms file to Areté format (adds Summary, Action Items)
```

**Templates in this plan** document the *final format* after both stages complete. They do NOT replace core adapter templates. Core adapters (`packages/core/src/integrations/{fathom,krisp}/index.ts`) continue using their hardcoded `DEFAULT_TEMPLATE`.
