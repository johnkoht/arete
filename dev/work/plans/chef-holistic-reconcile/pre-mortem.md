## CRITICAL risks — expanded

### 1. Approval-volume explosion + auto-approval flood (SP)

**Problem**: The benchmark celebrates 4–5× recall but the human approval funnel is unchanged. Two failure branches, both observed-adjacent in code today: (a) `processMeetingExtraction` auto-approves `confidence > 0.8` and the backend logs "Auto-approved N high-confidence items" (`agent.ts:335–340`) — a verbose frontier model emitting high confidence on 100 items/day mass-creates commitments, worsening the exact commitment-rot baseline (113 open) this work is supposed to help; (b) if everything lands `pending`, John triages it manually and the winddown gets *slower*, which for the primary daily user is project-killing regardless of recall numbers.

**Mitigation** (fold into SP W1 + W5):
- W1: in `single_pass` mode, approval status derives from **tier, not confidence**: only `blocker` may auto-approve; `high` stages pending-first; `normal` and `uncertain` never auto-approve. Confidence floats become telemetry.
- W5: add **AC11 (approval-budget)**: across the 11-meeting corpus, auto-approved items ≤ legacy baseline +20%, and median *pending items requiring a decision per meeting* ≤ 12 after W4 collapse rules. Gate the flag flip on this alongside AC1–AC10.
- Soak metric: item-fates approval rate AND John's wall-clock per winddown vs the 6/9 baseline.

**Verification**: eval scorecard has an approval-funnel column; one winddown timed before/after.

### 2. Shadow soak measures the wrong world (CHR W7)

**Problem**: W7 keeps inline reconcile live while the engine shadows. But inline reconcile *mutates the input* at extract time (first-occurrence-wins skips, reverse-stamping) before the engine reads the day. The engine is therefore never tested on its real production input (pure extractions); agreement-rate ≥90% is nearly guaranteed and meaningless; arc-assembly and engine-only-catch metrics are structurally suppressed. W6 (irreversible deletion) is gated on this report.

**Mitigation** (rewrite W7):
- During the soak, the extract step **persists the pre-reconcile extraction output** (a `raw_staged` snapshot in frontmatter or a sidecar JSON) *before* inline reconcile mutates statuses. The shadow engine consumes the raw snapshot — the same shape it will see post-W6.
- Diff three-way: raw → inline decisions vs raw → engine decisions. Now agreement, engine-only, inline-only, and arc events are all measured on identical, uncontaminated input.
- Add to AC5: at least 2 soak days must contain a real cross-meeting duplicate and at least 1 a same-day supersession arc (if the calendar doesn't provide them, replay a synthetic day) — otherwise the soak passes vacuously on boring days.

**Verification**: shadow log entries reference raw-snapshot item IDs, not post-reconcile statuses; soak report shows nonzero arc-assembly events.

### 3. Partial-ship seam between the plans (SP sequencing + CHR gate)

**Problem**: Two windows of damage. (a) SP's gate "flips the default mode" at W5 but W4 (tier-sorted view, normal-collapse) lands *after* the gate — production winddowns render uncapped, unranked staging for some number of days. (b) CHR assumes tiers/⚠/continuation_of/direction:none exist; SP's rollback is "flip `extraction_mode: legacy`" — doing so mid-CHR-soak feeds the engine legacy-shaped items with no specified behavior, corrupting the soak or the winddown itself.

**Mitigation**:
- SP Sequencing: reorder to W1 → W2 → W5 (eval, flag stays legacy) → W3 + **W4** → **then** flag flip → soak. Make it an explicit AC: "default-mode flip requires AC9 (view ranking) green in the same release."
- CHR W1 (spec): add a **degraded-input contract** — engine treats missing tiers as `normal`, missing ⚠ as `uncertain: false`, missing continuation claims as "nominate everything"; one Layer-1 unit test feeds a legacy-shaped ledger.
- CHR hard gate: add "SP rollback during CHR soak ⇒ engine shadow auto-pauses and soak clock resets" to W7.

**Verification**: grep the SP plan for the reordered sequencing line; CHR Layer-1 test suite contains the legacy-shape fixture.

## Verdict

**Ship-with-changes.** The core thesis is sound and well-evidenced (the cap-loss and mirror-pair mechanisms are real and verified in source; the benchmark design is honest about its confounds). But three changes are load-bearing — without them the plans fail at exactly the points they were written to fix:

1. **Fix W7's shadow design (CHR)**: persist pre-reconcile raw extraction snapshots and shadow on those — otherwise the soak report that gates the irreversible W6 deletion measures a world that won't exist.
2. **Gate the SP flag flip on W4 + an approval-budget AC**: tier-derived approval (only blockers auto-approve) and a pending-items-per-meeting budget — otherwise 4–5× volume either floods commitments via confidence>0.8 auto-approval or floods John's evening, and the primary user churns on day one.
3. **Make silent drops impossible and audits empirical**: AC8 enumerates every drop point including the silent 0.65-confidence `continue`; the W1 direction audit is re-run by grep (the plan's file list is wrong, and Phase 9 already taught this lesson once).

Secondary but cheap: cut or respec Jira to the skill-gather pattern, keep the detectors as permanent drift telemetry, and decouple the weekly rewire from the engine's first production cycle.
