## 2026-03-03: build-agents.ts compression functions are hardcoded

**What broke**: Updated `.agents/sources/guide/intelligence.md` expecting `npm run build` to pick up changes in the compressed dist/AGENTS.md output. The changes didn't appear because `compressIntelligence()` returns a **static string** that ignores the source file content entirely.

**Why it matters**: Most compression functions in `build-agents.ts` are hardcoded — `compressIntelligence()`, `compressVision()`, `compressWorkspaceStructure()`, `compressWorkflows()` all return static strings. Only `compressCLICommands()`, `compressSkillsTable()`, and `compressRulesList()` actually parse their source files. When updating guide sources, you must also update the corresponding compression function.

**Affected functions**:
- `compressIntelligence()` — hardcoded, ignores intelligence.md
- `compressVision()` — hardcoded, ignores vision.md
- `compressWorkspaceStructure()` — hardcoded, ignores workspace-structure.md
- `compressWorkflows()` — hardcoded, ignores workflows.md
- `compressCLICommands()` — partially hardcoded (`tool_selection`, `scope`, `proactive` lines are static; command list is parsed)
- `compressSkillsTable()` — parses source file ✅
- `compressRulesList()` — parses source file ✅
- `compressMemorySection()` — parses source file ✅

**How to avoid**: When editing any `.agents/sources/` file, check the corresponding compression function in `scripts/build-agents.ts`. If the function is hardcoded, update both the source file AND the function. Verify by running `npm run build` and checking `dist/AGENTS.md` for the expected changes.

## 2026-03-31: Root AGENTS.md overwrite with GUIDE content

**What broke**: Root AGENTS.md (BUILD mode, hand-written) was overwritten with dist/AGENTS.md content (GUIDE mode, generated). This happened after commit acbc1e9 removed the `dev` target from build-agents.ts, but subsequent commits (e.g., c57e944 "regenerate AGENTS.md") still modified root AGENTS.md with generated content — likely via manual copy from dist/AGENTS.md.

**Why it matters**: Root AGENTS.md and dist/AGENTS.md serve different purposes:
- `AGENTS.md` (root): Hand-written, BUILD mode — contains [Identity], [Expertise], [Roles], build-specific skills
- `dist/AGENTS.md`: Generated, GUIDE mode — contains shared + guide content for installed workspaces

Overwriting root with dist content removes BUILD-specific context like [Identity], [Expertise], [Roles] sections that subagents need.

**Detection**: The generated header `<!-- Generated by build-agents.ts -->` combined with `guide/` source file references indicates GUIDE content. BUILD AGENTS.md should have `> What is available — system awareness for BUILD mode agents.`

**Safeguard added**: `checkRootAgentsIntegrity()` function now runs before every prod build. It:
1. Checks if root AGENTS.md has generated header + GUIDE source refs (BAD)
2. Checks for BUILD-specific markers like [Identity], [Expertise] (GOOD)
3. Fails the build with clear instructions if root AGENTS.md is corrupted

**How to avoid**:
- Never copy dist/AGENTS.md to root AGENTS.md
- Edit root AGENTS.md directly for BUILD mode changes
- The build will now fail with instructions if this happens again

**To restore**: `git show cd640c4:AGENTS.md > AGENTS.md` (or any commit before c57e944)
