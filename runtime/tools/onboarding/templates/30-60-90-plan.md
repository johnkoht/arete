## Overview

This plan guides your onboarding through three phases: Learn, Contribute, and Lead. Each phase builds on the previous, moving you from newcomer to established contributor.

---

## Phase 1: Learn (Days 1-30)

**Goal**: Build deep understanding of the business, product, users, and team

**Theme**: Curiosity over contribution

### Week 1: Orient

- [ ] Complete HR/admin onboarding
- [ ] Set up all tools and access
- [ ] Meet with manager to align on expectations
- [ ] Read company overview docs (wiki, handbook)
- [ ] Schedule initial 1:1s with immediate team

### Week 2: Explore

- [ ] Deep dive into the product (use it extensively)
- [ ] Read key PRDs and strategy docs
- [ ] Begin stakeholder mapping
- [ ] Start drafting business-overview.md
- [ ] Identify key questions to answer

### Week 3: Connect

- [ ] 1:1s with cross-functional partners
- [ ] Shadow customer calls or review recordings
- [ ] Start drafting products-services.md
- [ ] Start drafting users-personas.md
- [ ] Document initial observations

### Week 4: Synthesize

- [ ] Consolidate learnings from weeks 1-3
- [ ] Complete draft context files
- [ ] Identify gaps in understanding
- [ ] Start thinking about where you can contribute
- [ ] Week 4 check-in with manager

**Phase 1 Success Criteria**:
- [ ] Can explain the business model clearly
- [ ] Have met with all key stakeholders (15+ 1:1s)
- [ ] Draft context files exist for core areas
- [ ] Clear understanding of current priorities
- [ ] Open questions documented

---

## Phase 2: Contribute (Days 31-60)

**Goal**: Start adding value while continuing to deepen understanding

**Theme**: Helpful contributor

### Week 5-6: Engage

- [ ] Join an active project in supporting role
- [ ] Take on first small, scoped task
- [ ] Second round of 1:1s with key stakeholders
- [ ] Deepen understanding of competitive landscape
- [ ] Refine context files based on new learnings

### Week 7-8: Deliver

- [ ] Complete first deliverable/contribution
- [ ] Document your first "win"
- [ ] Share observations and strategic questions
- [ ] Identify target area for Phase 3 ownership
- [ ] Week 8 check-in with manager

**Phase 2 Success Criteria**:
- [ ] Delivered at least one meaningful contribution
- [ ] Key stakeholders see you as helpful and competent
- [ ] Have clear target for Phase 3 ownership
- [ ] Context files are mostly complete
- [ ] Building reputation as thoughtful contributor

---

## Phase 3: Lead (Days 61-90)

**Goal**: Own something meaningful and establish your presence

**Theme**: Confident owner

### Week 9-10: Own

- [ ] Take ownership of identified initiative
- [ ] Begin leading (not just supporting)
- [ ] Propose improvements based on learnings
- [ ] Expand cross-functional relationships
- [ ] Continue refining context files

### Week 11-12: Establish

- [ ] Drive initiative to meaningful milestone
- [ ] Become "go-to" person for your area
- [ ] Finalize all context files
- [ ] Document outcomes and wins
- [ ] Develop 6-month vision

### Graduation

- [ ] Review all outputs for completeness
- [ ] Complete graduation criteria checklist
- [ ] Manager sign-off / feedback session
- [ ] Promote context files to main context/
- [ ] Archive onboarding project

**Phase 3 Success Criteria**:
- [ ] Successfully led at least one initiative
- [ ] People seek you out for expertise
- [ ] Context files complete and accurate
- [ ] Confident operating independently
- [ ] Clear vision for ongoing focus

## Questions to Answer

### About the Business
- [ ] What problem does the company solve?
- [ ] What's the business model?
- [ ] Who are the key customers?
- [ ] What's the competitive landscape?

### About the Product
- [ ] What's the product strategy?
- [ ] What are the current priorities?
- [ ] What's the roadmap?
- [ ] What's technical architecture (high level)?

### About the Team
- [ ] How are decisions made?
- [ ] What are the unwritten rules?
- [ ] Who has influence?
- [ ] What's the culture really like?

### About Your Role
- [ ] What does success look like?
- [ ] What are the unstated expectations?
- [ ] Where can you add unique value?
- [ ] What should you avoid?
